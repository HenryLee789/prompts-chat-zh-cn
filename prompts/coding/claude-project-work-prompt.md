# 克劳德 - 项目工作提示词

## 中文说明

用于让 AI 围绕「克劳德 - 项目工作提示词」执行编程与技术实现任务。它会保留原始角色、任务目标、约束条件和输出要求，适合直接复制给 ChatGPT、Claude、Gemini 等对话式 AI 使用。

## 使用场景

* 代码解释、debug、review 和 refactor
* 生成技术方案、测试用例或实现步骤
* 围绕 API、JSON、CLI、React、TypeScript、Node.js 等技术任务给出可执行指令
* 围绕 Claude 等主题生成结构化结果

## 适用人群

* 程序员
* 技术负责人
* 代码学习者
* 开发者

## 中文 Prompt

```md
在进行任何编辑之前，计划重新设计此网页。目标：
提高视觉层次、清晰度、信任度和conversion rate
同时保留当前的技术堆栈。你的流程：
1. 检查现有的代码库、组件、样式、标记和布局基元。
2. 确定当前实施中的 UX/UI 问题。
3. 如果品牌/风格/转换意图不清楚，请提出澄清问题。
4. 在 Markdown 中制定设计优先的实施计划。包括：
- 当前状态审计
- 主要可用性和视觉设计问题
- 拟议的信息架构
- 逐节页面计划
- 元件库存
- 重用、扩展、创建决策
- 需要更改设计令牌
- React行为笔记
- 无障碍考虑
- 分步实施顺序
- 风险和悬而未决的问题

限制条件：
- 尽可能重用现有组件
- 保持设计系统的一致性
- 暂不实施

约束条件：
- 严格保留原 prompt 中的限制条件、禁止事项和输出边界。
```

### Prompt

```md
Plan a redesign for this web page before making any edits.

Goal:
Improve visual hierarchy, clarity, trust, and conversion
while keeping the current tech stack.

Your process:
1. Inspect the existing codebase, components, styles, tokens, and layout primitives.
2. Identify UX/UI issues in the current implementation.
3. Ask clarifying questions if brand/style/conversion intent is unclear.
4. Produce a design-first implementation plan in markdown.

Include:
- Current-state audit
- Main usability and visual design issues
- Proposed information architecture
- Section-by-section page plan
- Component inventory
- Reuse vs extend vs create decisions
- Design token changes needed
- Responsive behavior notes
- Accessibility considerations
- Step-by-step implementation order
- Risks and open questions

Constraints:
- Reuse existing components where possible
- Keep design system consistency
- Do not implement yet
```
