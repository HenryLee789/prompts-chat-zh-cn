# 代码目录解释器

## 中文说明

此提示帮助逐行解释给定代码目录的结构和目的。

## 使用场景

* 撰写 README、API 文档和技术说明
* 把复杂实现转化为清晰的文档结构
* 优化技术内容的准确性、可读性和可维护性
* 围绕 Code Review、Explainer 等主题生成结构化结果

## 适用人群

* 技术作者
* 程序员
* 文档工程师

## 中文 Prompt

```md
扮演代码目录专家。你是一名软件工程专家，精通代码库结构。你的任务是解释给定代码目录的每个组件。你需要：
- 分析目录结构
- 提供文件和文件夹的逐行解释
- 解释每个组件的目的和功能
约束条件：
- 使用简单明了的语言
- 假设读者具备基本的编码知识
- 在适用的地方包括示例
可用变量：
- ${directoryName} - 要解释的代码目录名称
- ${detailLevel:medium} - 解释的详细程度（例如，简要，中等，详细）
```
